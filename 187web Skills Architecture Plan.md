# **187web Ecosystem: Claude Code Skills Architecture**

*Concept: "Killer Web" — Excellent, well-thought-out, precise, and highly effective execution.*

## **1\. Brand Identity & Aesthetic**

* **Mascot:** The Black Widow (Stylized, digital representation of lethal precision).  
* **Color Palette:**  
  * **Base:** Abyssal Black (\#080808) to Dark Grey (\#1A1A1A).  
  * **Structure:** Steel Grey (\#4A4A4A) and Ash (\#CCCCCC).  
  * **Critical/Alerts:** Widow Red (\#FF0000).  
  * **Success/Execution:** Toxic Avenger Green (\#39FF14).  
* **Vibe:** Lethal efficiency, highly calculated, no wasted overhead.

## **2\. The Central Nervous System: Obsidian "Local Brain" Integration**

The 187web ecosystem does not exist in a vacuum; its "web" is your Obsidian Knowledge Graph. Powered by **Claudian** (Claude API \+ Obsidian API integrations), the vault serves as the persistent memory, context engine, and command center.

* **The Vault as State:** All skills read their context from active Obsidian notes and output their results as dynamically linked markdown files.  
* **Metadata Routing:** YAML frontmatter and tags (\#ml-architecture, \#drafting) automatically dictate which 187web skill or persona is summoned.

## **3\. Skill Decomposition & Integration Matrix**

Here is how the foundational AI tools, personas, and architectures operate as highly focused skills directly within the Obsidian/Claudian brain.

### **Skill 0: 187repo-eval, 187free, 187research (Suite Expansion)**

* **187repo-eval:** Gate for evaluating third-party repos, packages, and tools for adoption into the 187web suite. Produces a scored Markdown scorecard with an `adopt` / `adopt-with-guardrails` / `watch` / `reject` recommendation.
* **187free:** No-cost solution engine for free-tier, open-source, local-first, and public-API tooling across hosting, databases, auth, AI, accessibility, and client-budget projects.
* **187research:** Research-grade lab engine for scholarly/biomedical/mathematical/software source routing, evidence classification, reproducible labs, and publication-ready artifacts. Always pairs with 187free for no-cost lab hosting.
* **Obsidian Integration:** A portable vault pack lives under `docs/187suite-vault/` and can be copied into any Obsidian vault as `_system/`, `_templates/`, and dashboards.

### **Skill 1: widow-weaver (The Core Prompt & Vault Engine)**

* **187web Function:** An AI skill dedicated to spinning complex, highly-tuned system prompts and seamlessly integrating the output into the Obsidian vault.  
* **Obsidian Integration:** Reads standard template files from a hidden /.187web/prompts folder in the vault.  
* **Integrated Prompt Library (The Web Strands):**  
  * *Document Summarization:* Extracts arguments from vault PDFs/long-notes; outputs concise, backlinked bullet points.  
  * *Linguistic Translation:* Preserves markdown formatting and technical nuance perfectly across notes.  
  * *Tone Polishing:* Rewrites active notes for a professional, assertive tone directly via Obsidian API.  
  * *PDF Dialogue:* Active file context mapping (chatting with local vault PDFs).  
  * *Office/Corporate:* Email Follow-Up, Task Extraction (auto-generates Obsidian task lists \- \[ \]).  
* **Output Vibe:** Dense, perfectly structured Markdown highlighted with **Toxic Green** callouts directly injected into the active Obsidian pane.

### **Skill 2: neuro-toxin (The Inference & Context Tuner)**

* **187web Function:** Configures local LLM/API parameters dynamically based on what part of the "brain" (vault) you are working in. Controls the "toxicity" (variance) and "lethality" (precision).  
* **Obsidian Integration:** Reads YAML frontmatter (e.g., toxicity: high for brainstorming notes, lethality: max for code notes) to adjust API calls on the fly.  
* **Integrated Tuning Parameters:**  
  * top\_k (Default: 40): Lowering increases "lethality" (focused/conservative for factual notes).  
  * temperature (Default: 0.8): 0.0 \= Absolute precision (Greedy, for Reference notes); 1.5 \= Maximum variance (for Daily Notes/Ideas).  
  * mirostat (Range: 0-2): Dynamic perplexity adjustment to avoid cognitive traps during long generative sessions.  
* **Output Vibe:** Claudian API request logs in the terminal show sliders in **Grey and Red**, reacting instantly to file switching in Obsidian.

### **Skill 3: silk-sandbox (The Execution & Containment Engine)**

* **187web Function:** Derived from the **OpenClaw** architecture. A secure, isolated environment for running code generated by Claude *directly from Obsidian code blocks*.  
* **Obsidian Integration:** Allows you to click a "Run" button on any \`\`\`python or \`\`\`bash block in Obsidian. The code is wrapped in "silk" and sent to an isolated container.  
* **Integrated Architecture:**  
  * Secure execution within isolated subsystems (e.g., Docker/Crostini).  
  * Results are appended back into the Obsidian note beneath the code block.  
* **Output Vibe:** Standard output text injected as **Grey** blockquotes. Detected security breaches trigger flashing **Widow Red** alerts in the Obsidian UI.

### **Skill 4: web-crawler (The Agentic Navigator & Clipper)**

* **187web Function:** Derived from the **Daneel** framework. An autonomous agent that traverses web interfaces to gather context and weaves it directly into the Local Brain.  
* **Obsidian Integration:** Bypasses standard web clippers. It autonomously navigates stateful web apps, extracts targeted DOM data, and creates perfectly formatted, fully tagged .md files in your vault.  
* **Integrated Agentic Capabilities:**  
  * *Web Automation:* Browses, extracts data, and resolves queries initiated from Obsidian commands.  
  * *Knowledge Weaving:* Automatically generates Wikilinks (\[\[Like This\]\]) based on existing vault architecture to ensure new data connects to the Local Brain immediately.  
* **Output Vibe:** Background terminal logs trace its path with **Toxic Green** success pings upon finding target data and saving the .md file.

### **Skill 5: swarm-mind (The Engineering Personas)**

* **187web Function:** A routing skill that injects heavy, expert-level system instructions into the Claudian session based on vault metadata.  
* **Obsidian Integration:** Triggered by specific vault directories or tags. Opening a note in /Projects/MachineLearning automatically summons the ML Architect spider.  
* **Integrated Personas:**  
  * *ML Systems Architect:* Data pipelines, LoRA, drift monitoring.  
  * *LLM Deployment Architect:* RAG, vLLM frameworks.  
  * *Multimodal Agent Designer:* GUI automation.  
  * *Edge AI Specialist:* NPU offloading.  
  * *Industry Specialists:* Finance, Space Economy, Biotech, Web3.  
* **Output Vibe:** A subtle UI indicator in Obsidian (subdued **Black/Grey**) that shows which persona is currently "active" and reading the active pane.

## **4\. Deployment Strategy**

* **Phase 1: The Brain Connect (Claudian \+ widow-weaver)** (Establish the bi-directional sync between the Claude API and the Obsidian Markdown vault).  
* **Phase 2: Contextual Tuning (neuro-toxin & swarm-mind)** (Map YAML frontmatter and vault tags to dynamic inference parameters and personas).  
* **Phase 3: Active Execution (silk-sandbox & web-crawler)** (Allow the brain to execute code locally and autonomously forage the web for new data).
