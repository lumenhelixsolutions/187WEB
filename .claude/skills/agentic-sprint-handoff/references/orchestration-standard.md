# Agentic Orchestration Standard

One program orchestrator owns the integration branch, dependency ordering, shared files, phase gates, final validation, and pull request.

Every subagent receives one role, one objective, an explicit file allowlist, prohibited files, dependencies, token budget, required artifacts, acceptance commands, and a structured return report.

Parallelize only independent lanes. Serialize README, AGENTS, registries, aliases, command tables, release files, package metadata, and public indexes.

Retries are bounded and recorded. Failed validation blocks the phase gate. FUSE records conflicts, selected evidence, rejected options, and uncertainty.
