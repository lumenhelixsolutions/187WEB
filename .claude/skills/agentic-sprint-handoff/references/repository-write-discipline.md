# Repository Write Discipline

1. Verify repository, default branch, current HEAD, status, and permissions.
2. Create a feature branch for substantial work.
3. Audit before implementation.
4. Assign non-overlapping file ownership.
5. Prefer small, reviewable patches.
6. Re-read the diff before every commit.
7. Modify canonical generators instead of hand-editing generated output.
8. Preserve unrelated work.
9. Never weaken validation.
10. Gate destructive or irreversible operations.
11. Produce dependency-ordered commits.
12. End with a PR and rollback contract.
