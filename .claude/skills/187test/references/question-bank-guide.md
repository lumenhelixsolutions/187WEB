# Question Bank Guide

Use this reference when 187TEST writes quiz, test, or placement items.

## Item quality rules

1. **One idea per item.** Each question measures a single skill or concept.
2. **Clear stem.** The question is complete without reading the options.
3. **Plausible distractors.** Wrong answers look reasonable to a learner who has not mastered the concept.
4. **No trick wording.** Avoid "all of the above" unless it is the only correct pattern.
5. **Consistent grammar.** Options match the stem in number and tense.
6. **Avoid negatives.** If you must use "not" or "except," bold it.

## Item types

| Type | Best for | Notes |
|---|---|---|
| Multiple choice | Recall, concept recognition, troubleshooting | Limit to 3–5 options |
| Checkbox | Multi-select procedures, feature lists | State "select all that apply" |
| True/False | Quick checks only | Easy to guess; use sparingly |
| Short answer | Definitions, commands, naming | Needs human or regex scoring |
| Code snippet | Syntax, debugging, output prediction | Test in actual runtime |
| Matching | Relationships, terms to definitions | Keep lists short and grouped |

## Difficulty calibration

- **Beginner:** item requires recognition or direct recall.
- **Intermediate:** item requires application in a familiar context.
- **Advanced:** item requires synthesis, debugging, or transfer to a new context.

Use at least one item per stated learning outcome. Spread difficulty across the test rather than clustering hard items at the end.

## Feedback quality

- Correct answer explanation: why it is right.
- Distractor explanation: why each common wrong answer is wrong.
- Link to resource for further study.

## Accessibility

- Avoid color-dependent items.
- Avoid items that rely on fine motor skills unless alternatives exist.
- Use plain language; define jargon in the stem.
- Provide sufficient time or allow pausing.
