# 187SEO Analytics Standard

## Search Console

- Verify ownership for all domain variants (www/non-www, HTTP/HTTPS).
- Monitor Performance report: queries, pages, countries, devices, search appearances.
- Review Indexing report for errors, warnings, valid pages, and exclusions.
- Track Core Web Vitals, Mobile Usability, and Enhancements reports.
- Set up sitemap submission and monitor coverage.

## GA4

- Confirm GA4 property linked to Search Console where possible.
- Track organic sessions, engaged sessions, conversions, and key events.
- Build exploration reports for landing pages and user journeys.
- Use attribution reports to understand SEO's contribution alongside other channels.

## KPI definitions

| KPI | Definition |
|-----|------------|
| **Organic clicks** | Clicks from Google Search results |
| **Impressions** | Times a page appeared in search results |
| **Average CTR** | Clicks / impressions |
| **Average position** | Mean ranking position for tracked queries |
| **Organic conversions** | Goal completions attributed to organic search |
| **Core Web Vitals pass rate** | Percentage of URLs passing all three CWV metrics |
| **Indexed pages** | URLs reported as valid in Search Console |

## Reporting cadence

- Weekly: quick health check on clicks, impressions, and indexing errors.
- Monthly: detailed performance review, content refresh priorities, technical issues.
- Quarterly: strategy review, competitive benchmark, roadmap update.

## Anomaly triage

1. Confirm the anomaly in multiple data sources (Search Console, GA4, rank tracker).
2. Correlate with known changes: releases, migrations, algorithm updates, seasonality.
3. Identify affected pages and queries.
4. Form a hypothesis and implement a targeted fix.
5. Monitor recovery and document learnings.
