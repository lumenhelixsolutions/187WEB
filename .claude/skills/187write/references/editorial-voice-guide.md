# Editorial Voice Guide

Default voice and tone for 187WRITE outputs.

## Voice attributes

| Attribute | How we sound |
|---|---|
| Clear | Short sentences, plain words, no filler |
| Capable | Confident but never arrogant |
| Honest | Claims labeled, limits acknowledged |
| Inclusive | Respectful of identity, disability, and background |
| Pragmatic | Focused on what the reader can do next |

## Tone adjustments by channel

| Channel | Tone |
|---|---|
| README / docs | Direct, helpful, neutral |
| Landing page | Energetic, specific, honest |
| Newsletter | Conversational, informative |
| Social post | Punchy, friendly, no hype |
| Tutorial | Patient, encouraging, step-by-step |
| Course script | Engaging, clear, repeatable |
| API reference | Precise, scannable, minimal |

## What to avoid

- Superlatives without evidence ("best", "revolutionary", "guaranteed").
- Jargon without definition.
- Fear-based persuasion.
- Clickbait patterns.
- Exclamation-point overload.
- Gendered or ableist defaults.

## Pronouns and person

- Address the reader as "you".
- Refer to the product or team as "we" when appropriate.
- Use singular "they" as default for unknown individuals.

## Formatting preferences

- Sentence-case headings.
- Oxford comma: yes.
- Em dashes without spaces—like this.
- Code in backticks; file paths in backticks.

## Claim verbs

| Strength | Verbs |
|---|---|
| Strong | proves, demonstrates, measures |
| Moderate | suggests, indicates, models |
| Weak | implies, speculates, frames |

Route voice disputes to `187include` for identity safety and `187access-plus` for disability access.
