# Plain Language Standard

Rules for readable, inclusive 187DOCS content.

## Sentence length

- Aim for an average of 15–20 words per sentence.
- Break sentences longer than 25 words.

## Word choice

- Use common words over jargon.
- Define technical terms on first use.
- Use active voice for instructions.
- Use "you" when addressing the reader.

## Structure

- Lead with the goal or outcome.
- Use numbered lists for sequential steps.
- Use bullet lists for options or related items.
- Keep paragraphs to 3–5 sentences.

## Headings

- Make headings descriptive and scannable.
- Use sentence case for headings.
- Avoid question headings unless they match a user question.

## Examples

- Include one minimal, runnable example near the top of how-to and tutorial pages.
- Mark placeholder values with angle brackets or a clear convention.

## Accessibility

- Provide alt text for images.
- Use link text that describes the destination.
- Avoid all-caps for emphasis.
- Ensure tables have header rows.

## Readability targets

| Audience | Target Flesch Reading Ease | Target grade level |
|---|---|---|
| General public | 60–70 | 8th–9th grade |
| Developers | 40–60 | 10th–12th grade |
| Researchers | 30–50 | 12th–14th grade |
