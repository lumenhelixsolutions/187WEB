# Docs Audit Checklist

Use this checklist when 187DOCS audits a documentation set.

## Completeness

- [ ] README exists and answers: what, why, install, usage, contribute.
- [ ] Install guide covers all supported platforms.
- [ ] Quickstart delivers a first success in under 15 minutes.
- [ ] How-to guides exist for the top 5 user goals.
- [ ] Reference docs cover commands, APIs, schemas, and config.
- [ ] Explanation docs cover architecture and key decisions.
- [ ] Troubleshooting section addresses common errors.

## Accuracy

- [ ] Commands run as documented in a clean environment.
- [ ] Version numbers match the latest release.
- [ ] Screenshots or examples match current UI.
- [ ] Links resolve without 404s.
- [ ] External dependencies match current versions.

## Clarity

- [ ] Headings follow a clear hierarchy.
- [ ] Each page has one primary audience.
- [ ] Plain-language score is appropriate for audience.
- [ ] Acronyms are defined on first use.
- [ ] Code examples are syntax-highlighted.

## Accessibility and inclusion

- [ ] Images have alt text.
- [ ] Color is not the only means of conveying meaning.
- [ ] Language is identity-safe and respectful.
- [ ] Form examples do not force binary gender fields.

## Sync

- [ ] README matches docs site landing page.
- [ ] AGENTS.md reflects current conventions.
- [ ] App pages match docs content.
- [ ] CHANGELOG links to migration notes.

## Output

Produce an audit report with:
- Pages reviewed
- Issues found by severity
- Suggested fixes
- Owner assignments
- Re-audit date
