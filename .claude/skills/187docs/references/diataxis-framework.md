# Diátaxis Framework Reference

The four documentation quadrants used by 187DOCS.

| Quadrant | Purpose | Verb | Example |
|---|---|---|---|
| Tutorials | Learning-oriented | Learn | "Getting started with 187repo" |
| How-to guides | Task-oriented | Achieve | "Deploy a Next.js site to GitHub Pages" |
| Reference | Information-oriented | Look up | "187repo CLI command reference" |
| Explanation | Understanding-oriented | Understand | "How 187web routing works" |

## How to choose

1. Is the reader studying? → Tutorial
2. Is the reader solving a specific problem? → How-to guide
3. Is the reader checking a fact? → Reference
4. Is the reader building conceptual understanding? → Explanation

## Quadrant rules

### Tutorials

- Assume the reader is a beginner.
- Provide a guaranteed path to first success.
- Keep steps minimal and ordered.
- Do not explain every option; focus on the learning journey.

### How-to guides

- Assume the reader knows the basics.
- Focus on a single goal.
- Include prerequisites and verification steps.
- Offer troubleshooting links.

### Reference

- Be complete, accurate, and scannable.
- Use tables, lists, and precise definitions.
- Avoid narrative or persuasion.

### Explanation

- Connect concepts to decisions the reader must make.
- Use analogies sparingly and label them as analogies.
- Distinguish design rationale from current behavior.

## Anti-patterns

- Mixing tutorial and reference in the same page.
- Adding marketing copy to reference docs.
- Using how-to pages for conceptual background.
