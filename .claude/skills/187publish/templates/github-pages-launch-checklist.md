# GitHub Pages Launch Checklist

Use this checklist when updating the GitHub Pages product showcase for a release.

## Content

- [ ] Hero headline reflects current suite positioning.
- [ ] Hero subhead includes the 16 first-class skills and 2 subskills.
- [ ] Each skill has a one-line description.
- [ ] Each skill tile links to its app route or docs page.
- [ ] Install command snippet is current.
- [ ] Changelog snippet links to full `CHANGELOG.md`.

## Visual assets

- [ ] Hero image or animation optimized.
- [ ] Skill icons are consistent and accessible.
- [ ] Screenshots include alt text.
- [ ] Color contrast meets WCAG AA (verified by 187ACCESS+).
- [ ] Reduced-motion alternative provided for animations.

## Accessibility

- [ ] Page has a unique `<title>` and `<h1>`.
- [ ] Heading hierarchy is logical.
- [ ] All interactive elements are keyboard reachable.
- [ ] Focus indicators are visible.
- [ ] Alt text provided for images.
- [ ] Color is not the only means of conveying information.

## Inclusion

- [ ] Copy reviewed by 187INCLUDE for identity-safe language.
- [ ] Examples include diverse names and pronouns.
- [ ] No stereotypes in imagery or scenarios.

## SEO

- [ ] Meta description present.
- [ ] Open Graph and Twitter card metadata set.
- [ ] Canonical URL set.
- [ ] Structured data included where appropriate.
- [ ] No doorway pages or thin content.

## Performance

- [ ] Images lazy-loaded and compressed.
- [ ] No render-blocking resources.
- [ ] Core Web Vitals within acceptable range.

## Links

- [ ] GitHub repo link works.
- [ ] Install docs link works.
- [ ] Each skill route link works.
- [ ] No broken internal or external links.

## Post-launch

- [ ] GitHub Pages build succeeded.
- [ ] Custom domain or default Pages URL resolves.
- [ ] Mobile layout tested.
- [ ] 187PUBLISH drift check re-run after deploy.
