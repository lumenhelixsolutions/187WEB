# Learning Design Framework

Use this reference when 187LEARN plans courses, workshops, study plans, or learning paths.

## Outcome-first sequencing

1. Define 1–5 measurable learning outcomes per module.
2. List the evidence that proves each outcome.
3. Design practice activities that produce that evidence.
4. Add content only after activities are defined.
5. Sequence from concrete → conceptual → applied.

## Lesson anatomy

| Section | Purpose | Approximate ratio |
|---|---|---|
| Hook | Why this matters | 5% |
| Prior knowledge check | Anchor to what learners know | 10% |
| Concept | Core idea in plain language | 25% |
| Demo / example | Concrete model | 20% |
| Practice | Learner does the work | 30% |
| Check / feedback | Confirm understanding | 10% |

## Format choices

- **Self-paced text:** best for reference lookup and revision.
- **Screencast:** best for tool workflows; always provide transcript.
- **Live workshop:** best for questions and collaboration; provide async fallback.
- **Microlearning:** best for spaced review; keep under 10 minutes.
- **Project-based:** best for transfer; include milestones and rubric.

## Memory and transfer

- Use worked examples for novices; fade them as expertise grows.
- Add retrieval practice, not just re-reading.
- Provide varied practice contexts to improve transfer.
- Give specific, actionable feedback.

## Accessibility baseline

- Captions and transcripts for all audio/video.
- Alt text for meaningful images; decorative images marked null.
- Consistent headings and reading order.
- Predictable pacing with optional extended time.
- Multiple formats for key content.
