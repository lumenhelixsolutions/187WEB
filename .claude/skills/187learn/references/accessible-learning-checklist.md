# Accessible Learning Checklist

Use this checklist during 187LEARN design to catch access barriers before content ships.

## Perception

- [ ] All videos have captions and transcripts.
- [ ] Audio content has transcripts.
- [ ] Images, diagrams, and charts have alt text or text descriptions.
- [ ] Color is not the only way meaning is conveyed.
- [ ] Text can be resized without breaking layout.

## Operation

- [ ] All activities are keyboard navigable.
- [ ] Timed activities can be paused or extended.
- [ ] Interactive elements have visible focus indicators.
- [ ] Drag-and-drop has keyboard alternatives.

## Comprehension

- [ ] Learning outcomes are stated in plain language.
- [ ] Jargon is defined or linked.
- [ ] Instructions are chunked and sequenced.
- [ ] Examples reflect diverse learners without stereotyping.
- [ ] Content provides multiple representations where helpful.

## Engagement

- [ ] Learners can choose preferred format where feasible.
- [ ] Pacing is flexible or clearly bounded.
- [ ] Notifications and reminders respect focus and consent.
- [ ] Collaborative activities have inclusive roles.

## Assessment

- [ ] Knowledge checks are accessible to screen readers.
- [ ] Practice tasks offer clear success criteria.
- [ ] Feedback is specific and actionable.
- [ ] High-stakes assessments include accommodation path.

## Identity and language

- [ ] Names and pronouns are respected in examples and forms.
- [ ] Disability is described with respectful, person-first or identity-first language as preferred.
- [ ] No real learner data is exposed in examples.
