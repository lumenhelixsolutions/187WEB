# 187SKILLS Integration Map

Integration points between the 187SKILLS split (187FREE + 187RESEARCH) and the rest of the Lumen Helix ecosystem.

## 1. Obsidian

**Role:** Knowledge base, linked notes, dashboards, claim/evidence archive.

**Integration:** YAML frontmatter, tags, wiki links, Bases/Dataview dashboards, templates.

## 2. Claudian

**Role:** In-vault agent interface.

**Integration:** Manual triggers, selected-note actions, plan mode, file-safe edits.

## 3. Claude Code

**Role:** Local code/research engineer.

**Integration:** CLAUDE.md rules, .claude/skills, shell only with approval, repo/lab generation.

## 4. MCPVault

**Role:** Safe vault bridge.

**Integration:** Expose vault to allowed MCP clients with frontmatter and file-safety discipline.

## 5. Obsidian Git

**Role:** Rollback and audit.

**Integration:** Commit baseline before big changes, branch for experiments, never destructive git.

## 6. Smart Connections

**Role:** Semantic retrieval over notes.

**Integration:** Find related research, sources, projects, inventions, accessibility patterns.

## 7. 187FREE

**Role:** No-cost stack/tool/source scouting.

**Integration:** Runs automatically inside build plans, lab plans, product specs, accessibility specs, and research-to-product workflows.

## 8. 187RESEARCH

**Role:** Research-grade source routing and lab generation.

**Integration:** Runs automatically inside literature review, source discovery, claim audit, reproducible labs, invention readiness, and product research workflows.

## 9. Lumen Helix/OIQ

**Role:** Applied research/product pipeline.

**Integration:** Research → evidence → lab/demo → spec → repo → launch → public claim review.

## 10. Public/Published Outputs

**Role:** Public pages, demos, papers, posts, specs.

**Integration:** No public release without disclosure-risk check, claim audit, citation review, accessibility review, and 187FREE deployment plan.
