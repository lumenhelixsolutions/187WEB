# 187FREE Privacy Checklist

Use this checklist whenever the project involves disability, legal, medical, civic, invention, client, or private research data.

## Defaults

- [ ] Prefer local-first or private-by-default tools.
- [ ] Avoid unnecessary third-party storage.
- [ ] Avoid hidden telemetry.
- [ ] Minimize data collection to what is strictly needed.

## Consent

- [ ] Explicit consent obtained before sharing data.
- [ ] Explicit consent obtained before tracking, analytics, or monitoring.
- [ ] Explicit consent obtained before reminders or notifications.
- [ ] Explicit consent obtained before caregiver access or support escalation.

## Sensitive domains

| Domain | Extra care |
|---|---|
| Disability / assistive tech | No surveillance; no hidden caregiver mode; consent-first. |
| Medical / health | No diagnosis/treatment advice; require professional review; HIPAA-aware choices. |
| Legal / civic | Records preservation; chain of custody; avoid scraping restricted systems. |
| Invention / IP | NDA-aware; avoid public repos for unpublished IP. |
| Client / commercial | Contract review; data-processing agreements; no shadow IT. |
| Private research | Anonymization plan; IRB/ethics review if human subjects. |

## Tool evaluation

- [ ] Where is data stored?
- [ ] Who can access it?
- [ ] Is it encrypted at rest and in transit?
- [ ] Can the user delete or export their data?
- [ ] Are there subprocessors or data-sharing clauses?
- [ ] Does the free tier sell or monetize data?

## Documentation

- [ ] Privacy notes captured in the recommendation.
- [ ] Sensitivity review included in output.
- [ ] Mitigations documented if third-party must be used.
