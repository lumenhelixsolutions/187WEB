# 187FREE Source Policy

## Primary source

- [free-for.dev](https://free-for.dev/)
- [ripienaar/free-for-dev](https://github.com/ripienaar/free-for-dev)

## Secondary source families

- Official free-tier pages
- Official API docs
- Open-source repos
- Public API directories
- Public datasets
- Open educational resources
- Open science databases
- Free browser APIs
- GitHub, npm, PyPI, and package registries
- Free accessibility and assistive technology projects
- Free documentation, testing, security, design, deployment, and monitoring services

## Verification rules

1. Prefer official sources over aggregator listings.
2. Cross-check free-tier terms on the official site before recommending.
3. Note the date checked; free tiers change.
4. Do not recommend tools whose free tier has expired or become misleading.
5. Avoid affiliate links; link to official docs.
6. If a tool requires an API key, mention how to obtain it safely (never paste keys in notes).

## Quality rule

Do not recommend a tool only because it is free. Recommend the free option most likely to actually work for the stated need.

## Citation format

When documenting a recommendation, include:

- Tool name
- Official URL
- Free-tier summary
- Date checked
- Limitations noted
- Privacy / telemetry notes
