# 187RESEARCH Public Claim Standard

## Public-claim risk levels

| Level | Definition | Required action |
|---|---|---|
| **low** | Well-supported, non-sensitive, general knowledge. | Citation review only. |
| **moderate** | Interpreted or inherited claim; some nuance required. | Add context, limitations, and sourcing. |
| **high** | Clinical, legal, civic, disability, invention, or safety-relevant claim. | Human review, disclaimer, and documented approval. |
| **blocked** | Unsupported, speculative, or potentially harmful claim. | Do not publish; downgrade or remove. |

## Required reviews before public release

- [ ] Claim audit against the evidence ladder.
- [ ] Citation review: every public claim has a resolvable source.
- [ ] Disclosure-risk check: no private, confidential, or identifiable information.
- [ ] Accessibility review: public page meets WCAG 2.2 AA.
- [ ] 187FREE deployment plan: hosting, domain, analytics, and form handling reviewed.
- [ ] Privacy review: telemetry, cookies, third-party services, and consent flows documented.
- [ ] Medical/legal/civic/disability review if applicable.

## Disclaimers

Add the appropriate disclaimer for the domain:

- **Medical / clinical:** "This is not medical advice. Consult a qualified professional."
- **Legal / civic:** "This is not legal advice. Verify against current official sources."
- **Financial / regulatory:** "This is not professional or regulatory advice."
- **Invention / patent:** "This does not constitute legal or patent advice."
- **Experimental / preliminary:** "These results are preliminary and have not been peer-reviewed."

## Human-review gate

High-risk public claims require:

1. Named reviewer.
2. Reviewer qualification note.
3. Review date.
4. Approval or conditional approval with required changes.
5. Documented rationale for any conditional elements.

## Reversal policy

If a public claim is later found to be unsupported or contradicted:

1. Update or retract the claim.
2. Add a dated correction note.
3. Notify downstream consumers if known.
4. Archive the previous version with a retraction marker.
